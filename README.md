1.
Виртуализация на основе ОС (контейнеры)  использует общие динамические библиотеки, общие страницы памяти.
Поэтому все ресурсы ОС доступны для работы. Это может дать например прирост быстродействия.
Разделение ресурсов осуществляется на уровне ОС. 

В паравирутализации госетвая ОС используют только ресурсы выделенные гипервизором на хосте.
Отсюда быстродействие гостевой ОС теоретически будет ниже из за ресурсов потребляемых гипервизором.
С другой стороны из за отсутствия прямого доступа к хосту гипервизора повышается уровень безопасности.
```
```
2.
- Для высоконагруженной  базы данных, чувствительной к отказу я бы выбрал физический сервер:
Потому, что он обеспечивает более высокую производительность, имееет меньше точек отказа в виде "доп"
прокладок в виде гипервизора.
- Для Java приложений - контейнеры.
Потому, что обеспечивает использование общих ресурсов между контейнерами, а также хорошую масштабируемость.
- Для Windows системы используемой Бухгалтерским отделом выбиру паравиртуализацию
Потому как высокая безопасность из за изолированной среды (к хосту не дотянутся), возможность создания резерных копий
и быстрого востановления системы, возможности клонирования, возможности оперативного изменения необходимой мощности 
для виртуалки.
- Для систем, выполняющих высокопроизводительные расчеты на GPU поставлю "физику"
Причины те же что указаны в случае БД - более высокие требования к быстродействию, меньше точек отказа.
